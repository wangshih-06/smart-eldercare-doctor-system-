### P-001 【启动】双击 start.bat 窗口闪退

**首次发现**：2026-07-05 17:40  
**状态**：✅ 已修复  
**模块**：启动 / 部署

#### 现象
双击 `scripts/start.bat` 后窗口立刻关闭，无法看到任何错误信息，系统无法启动。

#### 根因（多重）
1. `mvn` / `java` 不在系统 PATH → 执行失败
2. MySQL 未运行时脚本直接 `exit` → 窗口消失
3. Maven 启动占用主窗口，用户关闭窗口后进程被杀
4. 错误路径无 `pause`，窗口静默关闭

#### 修复
**文件**：[scripts/start.bat:78-105](file:///c:/Users/30816/Desktop/智慧医养医生服务系统1.0版本/scripts/start.bat#L78-L105)

**改动**：
- `where mvn` / `where java` 检查环境，缺失时显示友好提示 + `pause`
- MySQL 未运行改为"提示 + 询问是否继续"，不再强制退出
- Maven 改用 `start cmd /k` 在独立窗口启动，主窗口提示"不会停止应用"
- 所有错误路径统一加 `pause >nul`

#### 验证
双击 `start.bat dev` → 看到"检查运行环境..."输出 → Maven 在新窗口启动 → 主窗口保持打开。

#### 经验教训
- 所有面向用户的脚本**必须在所有错误路径加 `pause`**
- **绝对不能让窗口静默关闭**，用户会误以为"脚本坏了"
- 启动脚本应独立运行，不要占用主窗口

### P-002 【登录】验证码无法加载

**首次发现**：2026-07-05 17:48  
**状态**：✅ 已修复（环境问题）  
**模块**：登录

#### 现象
登录页加载时验证码图片空白，Console 报 500 错误，无法登录。

#### 根因分析

**表面根因**：[CaptchaController.java:40-41](file:///c:/Users/30816/Desktop/智慧医养医生服务系统1.0版本/src/main/java/com/medical/controller/CaptchaController.java#L40-L41) 写入 Redis 时抛出 `RedisConnectionFailureException` → 500 错误 → 前端无法解析 base64 图片。

**真正根因**：**Redis 服务没启动**。90% 的"代码报错"实际上是环境问题。

#### 修复（环境层面）
- 启动 Redis：双击 `redis-server.exe` 或运行 `scripts/start.bat`（自动启动）
- **Cygwin 编译的 Redis 必须保持 PowerShell 窗口打开**，关闭窗口会终止进程
- 生产环境用 `RedisService.exe` 注册 Windows 服务

#### 验证
```bash
netstat -ano | findstr 6379
# 应输出：TCP 0.0.0.0:6379 LISTENING
```

#### 经验教训
- Redis 验证码依赖 → **必须先启动 Redis 再启应用**
- Cygwin 编译的 Redis 不支持原生 `--service-install` 参数
- PowerShell 窗口关闭 = Redis 进程终止，团队务必注意
- 收到"功能挂了"反馈，**先看环境**（服务 / 配置 / 网络），不要一上来就改代码

### P-004 【环境】MySQL 客户端在 PowerShell 闪退

**首次发现**：2026-07-05 17:46  
**状态**：✅ 已修复  
**模块**：环境 / 工具

#### 现象
在 PowerShell 运行 `mysql -u root -p` 后窗口立刻关闭，无法执行 SQL。

#### 根因
`C:\Program Files\MySQL\MySQL Server 8.0\bin` 不在系统 PATH，PowerShell 找不到 `mysql` 命令。

#### 修复（三种方案）

**临时方案**：
```powershell
$env:Path += ";C:\Program Files\MySQL\MySQL Server 8.0\bin"
mysql -u root -p
```

**永久方案**：
- 系统属性 → 环境变量 → Path → 添加上述路径

**PowerShell profile**：
```powershell
# 编辑 $PROFILE，加入：
$env:Path += ";C:\Program Files\MySQL\MySQL Server 8.0\bin"
```

#### 经验教训
- PowerShell 临时加 PATH：`$env:Path += ";xxx"`
- 也可以用 `& "C:\...\mysql.exe"` 直接调用可执行文件
- 团队新成员上手时**检查 PATH 配置**

- 修复：对齐 0→待接收 / 1→已接收 / 2→处理中 / 4→已拒绝；详情弹窗补全紧急程度、转入科室、预留床位

### O-001 【仪表盘】统计图表加载慢（5-10 秒）

**首次发现**：2026-07-05 18:30  
**状态**：📋 已入档，待 v1.1  
**模块**：仪表盘 / 性能

#### 现象
登录后进入主页，以下 3 个统计图表加载耗时 5-10 秒：
- **老人性别分布**
- **用户角色分布**
- **业务模块累计数据**

期间页面卡顿，ECharts 渲染慢。

#### 根因（初步分析）
- 三张图均调用后端聚合统计接口（COUNT / GROUP BY）
- 当数据量大时，全表扫描导致接口慢
- 仪表盘首屏未做"懒加载 / 骨架屏 / 异步并行"，3 个图依次请求后顺序渲染

#### 当前影响
- 仅首屏进入时有感知，不影响核心医疗功能
- 医生实际查档 / 录数据 / 干预等流程不受影响

#### 暂不处理的原因
- 按"开发标准"第 1 条：**仅修复直接影响功能正常运行的明显 bug，性能优化暂不处理**
- 三张图非核心医疗功能，仅做信息聚合
- 优化方案涉及：聚合 SQL 改写 / 缓存层（Redis）/ ECharts 异步渲染 / 骨架屏——工作量较大
- 需团队统一规划后再启动，**避免过度设计**

#### 待优化方向（v1.1 启动时参考）

| 方向 | 具体措施 |
|------|----------|
| 数据库索引 | elder_info 加 `idx_gender`，sys_user 加 `idx_user_type` |
| 缓存层 | Redis 缓存统计结果，TTL 5 分钟，定时刷新 |
| 前端并行 | 3 个图并行请求，各自独立 Loading 状态 |
| 骨架屏 | 首屏加载时用骨架占位，避免空白等待 |
| 后端优化 | 聚合接口加超时熔断，返回默认值 |

#### 触发处理的条件
- 团队达成"性能优化"阶段共识时启动
- 或线上用户反馈此问题成为主要痛点时启动

## 九、典型案例分析（教学用）

### 案例 1：Redis 加载困难（P-002）

**表面现象**：验证码无法显示  
**表面根因**：CaptchaController 写 Redis 失败  
**真正根因**：**Redis 服务没启动**

**为什么难以发现**：
- 报错信息指向 CaptchaController，让人误以为是代码问题
- 实际上 90% 的"代码报错"是**环境问题**（服务没启、配置错了、权限不够）
- 团队容易陷入"改代码"陷阱，忘了"先看环境"

**排查流程**：
```
1. 看日志报错 → 锁定出错的方法
2. 看该方法的依赖（Redis？DB？外部 API？）
3. 检查依赖是否启动（netstat / tasklist）
4. 检查配置是否正确（application.yml）
5. 启动 / 修正依赖后再试
6. 如果仍失败 → 才是代码问题
```

**给团队的建议**：
- 收到"功能挂了"反馈，**先看环境**（服务 / 配置 / 网络）
- 不要一上来就改代码
- [项目运行指南.md](file:///c:/Users/30816/Desktop/智慧医养医生服务系统1.0版本/docs/项目运行指南.md) 写明"先启动 MySQL → Redis → Spring Boot"

---

### 案例 2：浏览器缓存（P-010 + P-011 组合）

**表面现象**：前端代码改了，浏览器刷新后仍看不到  
**表面根因**：浏览器缓存了旧 JS  
**真正根因**：HTML 没版本号 + target/classes 没同步

**为什么难以发现**：
- 开发者用 IDEA 看的是源文件
- 浏览器看的是 target/classes
- 两者**可能不一致**
- 即使一致，浏览器还有自己的缓存层

**排查流程**：
```
1. 浏览器 F12 → Network 标签
2. 看 app.js 的状态码（200 还是 304）
3. 看 app.js 的 Size（应该跟磁盘一致）
4. 如果是 304 或 Size 偏小 → 浏览器缓存
5. Ctrl + F5 硬刷新
6. 如果还不行 → target/classes 没同步
7. 比较 src 和 target 文件大小 / 时间戳
```

**给团队的建议**：
- **改完代码立刻告诉用户 "请 Ctrl + F5"**
- 不要说"刷新一下"——普通 F5 不够
- HTML 必须带版本号（`?v=时间戳`）
- v1.1 引入 devtools 自动热部署

---

### 案例 3：忘记密码弹窗卡死（P-005）

**表面现象**：弹窗关闭后页面卡死  
**表面根因**：关闭逻辑只删了内容，没删外层

**为什么难以发现**：
- 弹窗"看起来"关了（内容消失了）
- 实际黑色遮罩还在，只是透明的
- 用户以为"系统卡了"

**排查流程**：
```
1. F12 → Elements 标签
2. 搜索"resetPwdModal"或"modal"
3. 看是否还有 DOM 节点残留
4. 如果有 → 关闭逻辑没删干净
```

**给团队的建议**：
- 弹窗关闭必须**删整个外层 DOM 节点**
- 不要只 `display: none`（会留节点）
- 不要只清 `.innerHTML`（会留外层）
- 用 Vue / React 的条件渲染（`v-if`）最稳

## 十、预防措施清单

| 问题类型 | 预防措施 |
|----------|----------|
| 启动脚本闪退 | 所有错误路径加 `pause` |
