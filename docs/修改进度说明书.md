## 三、详细修改说明

### 3.1 启动脚本重构（start.bat 重写）

**问题**：双击 start.bat 窗口闪退，无法看到错误。

**根因**：
1. mvn / java 不在系统 PATH
2. MySQL 未运行时脚本直接退出
3. Maven 启动占用主窗口，主窗口关闭后进程被杀
4. 错误路径缺少 `pause`，窗口直接消失

**修复**：
- `where mvn` / `where java` 检查环境，缺失时 pause
- MySQL 未运行改为"提示 + 询问是否继续"，不再强制退出
- Maven 启动改用 `start "Maven-SpringBoot-Dev" cmd /k`，独立窗口
- 所有错误路径统一加 `pause >nul`

**效果**：`双击 start.bat dev` → 看到清晰的状态输出 → 后端在新窗口跑 → 主窗口提示"按任意键关闭（不会停止应用）"。

### 3.2 验证码加载失败修复

**问题**：登录页验证码图片不显示，Console 报 500。

**根因**：[CaptchaController.java:40-41](file:///c:/Users/30816/Desktop/智慧医养医生服务系统1.0版本/src/main/java/com/medical/controller/CaptchaController.java#L40-L41) 写入 Redis 时 `RedisConnectionFailureException`。

**修复**：**环境问题，非代码问题**。启动 Redis（`redis-server.exe` 双击 / `scripts/start.bat` 自动启动）后即正常。

**教训**：Cygwin 编译的 Redis 用 `start "Redis" redis-server.exe` 命令行启动，**不要关掉 PowerShell 窗口**（进程会随窗口结束）。生产环境用 `RedisService.exe` 注册 Windows 服务。

### 3.8 api() catch 块变量作用域修复（B-015）

**问题**：点击"查看历史记录"后弹窗始终显示"暂无随访记录"，尽管后端 API 返回了正确的记录数据。

**根因**：`api()` 方法的 `catch` 块中引用了 `res` 变量（在 `try` 块中用 `const` 声明），作用域不匹配。当 fetch 请求失败时，`catch` 块抛出 `ReferenceError`，导致 `api()` 返回 `null`，`followRecordList` 始终为空数组。

**修复**（[app.js:2208-2210](file:///c:/Users/30816/Desktop/智慧医养医生服务系统1.0版本/src/main/resources/static/pages/app.js#L2208-L2210)）：
```javascript
// 改前
this.toast('提示', res?.msg || res?.message || '操作失败', 'error');
// 改后
this.toast('提示', e?.message || '操作失败', 'error');
```

**配合**：添加了 `console.log` 调试日志到 `openRecordListModal()` 方法，便于在浏览器 F12 控制台定位问题。

**经验**：
- `const` 声明的变量有块级作用域，`catch` 块中无法访问 `try` 块中的 `const` 变量
- `catch(e)` 应使用参数 `e` 获取错误信息
- 异步函数中添加调试日志有助于定位前端问题

### 3.9 随访计划 completed_count 数据同步（D1 步骤）

**问题**：随访计划列表中"已完成次数"与实际记录数严重不匹配。例如王大爷高血压随访显示 15/24，实际只有 8 条记录。

**根因**：种子数据通过 SQL 直接 INSERT 写入 `follow_record` 表，绕过了 `FollowUpServiceImpl.createRecord()` 中自动递增 `completed_count` 的逻辑。

**修复**：
- 执行数据库 UPDATE，以 `follow_record` 实际行数为准，同步全部 16 个计划的 `completed_count`
- 同时修正 `status`：当 `completed_count >= total_count` 时设为 2（已完成）

**影响计划**：
| 计划 | 修复前 | 修复后 | 实际记录 |
|------|:---:|:---:|:---:|
| 王大爷高血压 | 15 | 8 | 8 |
| 李奶奶糖尿病 | 6 | 8 | 8 → 状态变为「已完成」|
| 孙大爷高血压 | 19 | 1 | 1 |
| …其余 13 个 | 各有偏差 | 全部对齐 | — |

**经验**：
- 种子数据必须通过 Service 层写入，不能绕过业务逻辑
- 冗余字段（`completed_count`）应定期巡检修正，或改用视图动态计算

## 四、修改流程标准

> 团队所有成员请遵守。

### 4.1 改动前

1. **先看 [问题与优化总览.md](file:///c:/Users/30816/Desktop/智慧医养医生服务系统1.0版本/docs/问题与优化总览.md)**：避免重复造轮子
2. **明确改动类型**：Bug 修复 / 功能新增 / 配置变更 / 数据补充
3. **最小改动原则**：只改直接影响功能的部分，不做性能优化 / 重构

### 4.2 改动中

1. **前端文件改完必须同步到 `target/classes/`**（Spring Boot 直接读 classpath）
2. **HTML 引用必须带版本号**（`?v=YYYYMMDDHHMM`）
3. **数据库字段变更必须写迁移脚本**（`sql/xxx_xxx.sql`），不能用 ORM 自动迁移
4. **后端代码改完必须重启 Spring Boot**

### 4.3 改动后

1. **必须记录到 [修改记录.md](file:///c:/Users/30816/Desktop/智慧医养医生服务系统1.0版本/docs/修改记录.md)**：

## 十、关键问题诊断（2026-07-06）

### 10.1 "查看历史记录 → 暂无随访记录" 真因

**用户反馈**：点击"查看历史记录"后弹窗显示"暂无随访记录"。

**实际诊断步骤**：
1. ✅ 后端 API 验证：每个 plan 都有 5-6 条记录（如 planId=1 返回 5 条）
2. ✅ 前端代码验证：`openRecordListModal` / `openRecordDetail` 实现完整
3. ❌ **Spring Boot 进程不存在**（`tasklist` 找不到 `java.exe`）
4. ❌ 8080 端口**未被本应用监听**

**真正的根因**：
- **Spring Boot 没启动**！浏览器看到的"暂无随访记录"很可能是：
  - (a) 浏览器加载了旧的 `app.js`（缓存问题，C1.1 已用 `?v=` 解决）
  - (b) 后端宕机导致 API 401/500，弹窗误显示"暂无"
  - (c) 浏览器访问的是 8080 端口上**其他应用**（如英语六级词汇项目）

**用户必做的启动步骤**（**这是问题的关键**）：

```bash
# 步骤 1：启动 MySQL（已运行 ✅）

# 步骤 2：启动 Redis（必须）
cd "C:\Users\30816\Desktop\智慧医养医生服务系统1.0版本\Redis-8.8.0-Windows-x64-cygwin-with-Service"
start "" "redis-server.exe"
# 验证：netstat -ano | findstr 6379

# 步骤 3：启动 Spring Boot（必须）
# 方式 A（推荐）：在 IDEA 中打开项目，运行 MedicalDoctorApplication
# 方式 B：scripts/start.bat dev（在新窗口跑 Maven）
```

**启动后验证**：
```bash
# 健康检查
curl http://localhost:8080/api/auth/captcha
```
