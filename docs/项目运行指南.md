### 环境要求补充

| 环境 | 版本要求 | 说明 |
|------|---------|------|
| **MySQL** | 5.7+ 或 8.0+ | 需要提前安装并运行 MySQL 服务 |
| **Redis** | 7.x / 8.x | 需要提前启动 Redis 服务（项目根目录已内置 Redis） |
| **JDK** | Java 17+ | 推荐 JDK 17 或 JDK 21（兼容 Java 23） |
| **Maven** | 3.6+ | 用于编译和运行（项目自带 Maven Wrapper） |
| **操作系统** | Windows / macOS / Linux | 均可 |

> 本项目使用 **MySQL 数据库** + **Redis 分布式缓存**，启动应用前需确保两者均已运行。

### 数据库准备

```sql
-- 创建数据库（如尚未创建）
CREATE DATABASE IF NOT EXISTS medical_doctor 
  CHARACTER SET utf8mb4 
  COLLATE utf8mb4_unicode_ci;
```

### 数据库配置

```yaml
spring:
  datasource:
    driver-class-name: com.mysql.cj.jdbc.Driver
    url: jdbc:mysql://localhost:3306/medical_doctor?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true&createDatabaseIfNotExist=true
    username: root
    password: 自行填写
```

> 数据库账号密码可根据实际情况修改 `application.yml` 中的 `spring.datasource.username` 和 `spring.datasource.password`。

## 三、快速启动

### ⚠️ 前置步骤：启动 Redis 服务

本项目使用 **Redis 分布式缓存**，启动应用前必须先启动 Redis。Redis 已内置在项目根目录中：

```bash
# 1. 进入项目根目录
cd 智慧医养大数据公共服务平台医生服务系统

# 2. 启动 Redis 服务（保持此终端窗口运行）
Redis-8.8.0-Windows-x64-cygwin-with-Service\redis-server.exe
```

看到以下输出表示 Redis 启动成功：
```
* Ready to accept connections tcp
```

> Redis 默认端口为 **6379**，无密码。可在 `application.yml` 中修改配置。

---

### 方式一：Maven 命令启动（推荐开发使用）

> 请先确保 Redis 已在另一个终端中运行。

```bash
# 1. 进入项目根目录
cd 智慧医养大数据公共服务平台医生服务系统

# 2. 编译并启动应用（默认端口 8080）
mvn spring-boot:run
```

如需指定其他端口：
```bash
mvn spring-boot:run -Dspring-boot.run.arguments="--server.port=8088"
```

### 方式二：打包为 JAR 运行（推荐部署使用）

> 请先确保 Redis 已在另一个终端中运行。

```bash
# 1. 打包
mvn clean package -DskipTests

# 2. 运行 JAR
java -jar target/medical-doctor-service-1.0.0.jar
```

如需指定端口：
```bash
java -jar target/medical-doctor-service-1.0.0.jar --server.port=8088
```

### 方式三：IDE 启动

> 请先确保 Redis 已在终端中运行（见上方前置步骤）。

1. 使用 IntelliJ IDEA / Eclipse 打开项目
2. 等待 Maven 依赖下载完成
3. 运行 `com.medical.MedicalDoctorApplication` 的 `main` 方法

## 七、配置说明

### application.yml 关键配置

```yaml
server:
  port: 8080                          # 服务端口

spring:
  # Redis 分布式缓存配置
  redis:
    host: localhost                   # Redis 主机地址
    port: 6379                        # Redis 端口
    password:                         # Redis 密码（无密码则留空）
    database: 0                       # Redis 数据库索引
    timeout: 10000ms                  # 连接超时时间
    lettuce:
      pool:
        max-active: 20                # 连接池最大连接数
        max-idle: 10                  # 连接池最大空闲连接
        min-idle: 5                   # 连接池最小空闲连接
        max-wait: 5000ms              # 获取连接最大等待时间

  datasource:
    driver-class-name: com.mysql.cj.jdbc.Driver     # MySQL 驱动
    url: jdbc:mysql://localhost:3306/medical_doctor?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true&createDatabaseIfNotExist=true
    username: root                      # 数据库用户名
    password: 958888                    # 数据库密码
    type: com.alibaba.druid.pool.DruidDataSource
    druid:
      initial-size: 5                  # 连接池初始连接数
      min-idle: 5                      # 最小空闲连接
      max-active: 20                   # 最大活跃连接
      filters: stat,wall,slf4j         # 监控、防火墙、日志
  sql:
    init:
      mode: always                     # 每次启动自动执行 SQL
      schema-locations: classpath:db/schema.sql
      data-locations: classpath:db/data.sql
      continue-on-error: true          # 出错继续执行

jwt:
  secret: MedicalDoctorServiceSecretKey2024!@#$%
  expiration: 7200                    # Token 有效期（秒），默认 2 小时

mybatis-plus:
  global-config:
    db-config:
      logic-delete-field: deleted     # 逻辑删除字段
      logic-delete-value: 1
      logic-not-delete-value: 0
```

### 切换 H2 内存数据库（开发测试用）

如需切换为 H2 内存数据库（无需安装数据库），修改 `application.yml`：

```yaml
spring:
  datasource:
    driver-class-name: org.h2.Driver
    url: jdbc:h2:mem:medical_doctor
    username: sa
    password:
  h2:
    console:
      enabled: true
      path: /h2-console
```

同时需要在 `pom.xml` 中添加 H2 依赖（如尚未添加）：

```xml
<dependency>
    <groupId>com.h2database</groupId>
    <artifactId>h2</artifactId>
    <scope>runtime</scope>
</dependency>
```

### Druid 连接池监控

启动后可通过 `http://localhost:8080/druid` 访问 Druid 监控控制台：

| 参数 | 值 |
|------|-----|

| 登录密码 | `123456` |

可查看：数据源状态、SQL 监控、URI 监控、Session 监控等。

### SQL 初始化

每次启动时自动执行 `db/schema.sql`（建表）和 `db/data.sql`（初始化数据），通过以下配置控制：

```yaml
spring:
  sql:
    init:
      mode: always              # 每次启动执行（也可设为 embedded 或 never）
      continue-on-error: true   # 遇到重复插入等错误时继续执行
```

## 十、常见问题

### Q1: 端口被占用或数据库连接失败

```bash
# Windows 查看占用 8080 端口的进程
netstat -ano | findstr ":8080"
# 强制终止该进程（PID 替换为实际进程号）
taskkill /F /PID <进程号>

# 检查 MySQL 是否在运行
netstat -ano | findstr ":3306"
```

### Q2: 编译报错 Lombok 相关

确保使用 JDK 17+，并在 IDE 中安装 Lombok 插件。

### Q3: 登录后页面空白

1. 打开浏览器开发者工具（F12）→ 控制台
2. 查看是否有 JavaScript 报错（如缺少逗号等语法错误）
3. 清除浏览器 `localStorage`：`localStorage.clear()`
4. 重新访问 `http://localhost:8080`

### Q4: MySQL 连接失败

```bash
# 错误示例
com.mysql.cj.jdbc.exceptions.CommunicationsException: Communications link failure

# 排查步骤
1. 确认 MySQL 服务已启动：services.msc → 查找 MySQL
2. 确认用户名密码正确：mysql -u root -p
3. 确认数据库已创建：CREATE DATABASE IF NOT EXISTS medical_doctor;
4. 检查 application.yml 中的连接地址和端口
```

### Q5: 趋势图不显示

1. 确保先输入「老人ID」
2. 点击「生成模拟数据」生成测试数据
3. 在「监测指标」下拉框中选择要查看的指标
4. 点击「查询」刷新趋势图

### Q6: 验证码输入后提示错误

验证码区分大小写，点击验证码图片可刷新获取新的验证码。
